# 마크다운 기능

Docusaurus는 **[Markdown 문법](https://daringfireball.net/projects/markdown/syntax)**과 몇가지 **추가 기능**을 지원합니다.

## 메타데이터

마크다운 문서 맨위에 Front-Matter라는 메타데이터를 가지고 있습니다. [Front Matter](https://jekyllrb.com/docs/front-matter/):

```text title="my-doc.md"
// highlight-start
---
id: my-doc-id
title: My document title
description: My document description
slug: /my-custom-url
---
// highlight-end

## Markdown heading

Markdown text with [links](./hello.md)
```

## 링크

링크는 `[]()` 문법을 사용합니다.

```md

```md
Let's see how to [Create a page](/create-a-page).
```

```md
Let's see how to [Create a page](./create-a-page.md).
```

## 이미지

절대 경로를 사용하면 static 디렉토리에 있는 이미지를 참조할 수 있습니다. (`static/img/docusaurus.png`):

```md
![Docusaurus logo](/img/docusaurus.png)
```

![Docusaurus logo](/img/docusaurus.png)

현재 파일경로에서 부터 이미지를 참조할 수도 있습니다. 이는 특히 Markdown 파일과 관련된 이미지를 함께 사용할 때 유용합니다.

```md

```md
![Docusaurus logo](./img/docusaurus.png)
```

## 코드 블럭

Markdown 코드 블럭은 Syntax highlighting을 지원합니다.

    ```jsx title="src/components/HelloDocusaurus.js"
    function HelloDocusaurus() {
        return (
            <h1>Hello, Docusaurus!</h1>
        )
    }
    ```

```jsx title="src/components/HelloDocusaurus.js"
function HelloDocusaurus() {
  return <h1>Hello, Docusaurus!</h1>;
}
```

## 주의사항

Docusaurus는 주의사항과 콜아웃을 만들기 위한 특별한 문법을 가지고 있습니다.

    :::tip 팁

    이 기능을 쓰시오

    :::

    :::danger 조심

    이 기능은 위험합니다

    :::

:::tip 팁

## MDX and React Components

[MDX](https://mdxjs.com/)는 당신의 문서를 더 **interactive**하게 만들어주고 **Markdown 안에서 React 컴포넌트를 사용**할 수 있게 해줍니다.

```mdx title="hello.mdx"

```jsx
export const Highlight = ({children, color}) => (
  <span
    style={{
      backgroundColor: color,
      borderRadius: '20px',
      color: '#fff',
      padding: '10px',
      cursor: 'pointer',
    }}
    onClick={() => {
      alert(`You clicked the color ${color} with label ${children}`)
    }}>
    {children}
  </span>
);

This is <Highlight color="#25c2a0">Docusaurus green</Highlight> !

This is <Highlight color="#1877F2">Facebook blue</Highlight> !
```
